# Security policy

Thanks for helping keep Whether secure.

## Supported versions

Whether is currently pre-1.0 and maintained on the default branch.
Security fixes are prioritized for current `main`/default-branch code and current deployment artifacts.

## Reporting a vulnerability

Please **do not open a public GitHub issue** for suspected vulnerabilities.

Use one of these private channels instead:
1. **Preferred:** GitHub private vulnerability reporting ("Report a vulnerability" in the Security tab).
2. **Fallback:** Open a private maintainer contact request and label it `security-contact-needed`; maintainers will move follow-up to a private channel.

When reporting, include:
- A short description of the issue.
- Reproduction steps or proof-of-concept.
- Potential impact and affected surfaces.
- Any suggested remediation (if available).

## Response targets

We aim to:
- Acknowledge reports within 3 business days.
- Provide an initial triage response within 7 business days.
- Coordinate disclosure timing after a fix is available.

## Disclosure guidance

Please keep details private until we confirm a fix and disclosure window.
We appreciate responsible disclosure and can credit researchers in release notes unless anonymity is requested.
